# PeriPrint

Десктопное GUI-приложение для печати на термопринтерах Peripage (A6/A6+/A40/A40+) через Bluetooth. Перетащили файл — увидели точный чёрно-белый предпросмотр — напечатали в пару кликов, с сохранённым профилем принтера и историей.

**Статус:** рабочее приложение, проверено вживую на Peripage A40 (ALD-Y200). Linux — основная и единственная полностью поддерживаемая платформа.

## Установка и запуск

Проще всего — через AppImage, не требует Python, venv или `pip`:

1. Скачайте `PeriPrint-x86_64.AppImage` со страницы [Releases](https://github.com/RainManVays/perripage-ferrum/releases) репозитория.
2. Сделайте файл исполняемым и запустите:
   ```bash
   chmod +x PeriPrint-x86_64.AppImage
   ./PeriPrint-x86_64.AppImage
   ```
3. Только для печати по Bluetooth (один раз, системно):
   ```bash
   sudo apt install python3-bluez
   ```
   Без этого пакета приложение всё равно запустится и покажет предпросмотр — печать по Bluetooth просто останется недоступна, с понятным сообщением об ошибке вместо падения.

Если готового релиза ещё нет или вы на не-Debian-based дистрибутиве — соберите AppImage сами: `packaging/appimage/build.sh` (нужен только Docker). Подробности — в [docs/appimage-packaging-guide.md](docs/appimage-packaging-guide.md).

### Из исходников (для разработки)

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
python -m periprint.app
```

Bluetooth-транспорт (`peripage`/PyBluez) требует отдельной настройки системного пакета — см. [docs/BLUETOOTH_SETUP.md](docs/BLUETOOTH_SETUP.md).

## Первый запуск

1. Включите Bluetooth на принтере, нажмите «Сканировать» — приложение найдёт принтеры рядом (ищет устройства вида `PPG_<модель>_<id>`).
2. Выберите найденный принтер и подключитесь — профиль (имя, MAC, модель) сохранится для следующего раза.
3. Перетащите файл в окно (PNG/JPG/BMP, TXT или PDF) — появится предпросмотр в реальном разрешении принтера.
4. При необходимости настройте формат страницы (A4/A5/A6), поворот, зеркало, диапазон страниц, число копий — и нажмите «Печать».

## Возможности

- Bluetooth-скан и сохранённые профили принтеров.
- Drag & drop: изображения (PNG/JPG/BMP), текст, PDF (постранично).
- Точный предпросмотр 1:1 — дизеринг под фактическую ширину модели принтера.
- Импозиция: печать A5/A6-контента на A4-эквивалентном листе (2-в-1, 4-в-1), с автоповоротом под ячейку сетки и горизонтальным/вертикальным зеркалом.
- Диапазон страниц и число копий для PDF.
- Автонарезка длинных изображений на чанки с адаптивной паузой по % чёрных пикселей — защита от перегрева печатающей головки.
- Очередь печати с прогрессом по чанкам; реконнект и продолжение с места обрыва при разрыве Bluetooth-соединения.
- Отдельная вкладка «История» с завершёнными/неудачными заданиями.
- Живая реакция на статусы принтера (нет бумаги, открыта крышка, перегрев) — задание сразу ставится на паузу с понятной причиной, а не падает по таймауту.

Полный список требований, архитектура и план разработки — в [`periprint-spec.md`](periprint-spec.md); техническая документация (протокол принтера, найденные баги, решения) — в [`docs/`](docs/).

## Системные требования

- Linux x86_64 (проверено на Linux Mint/Cinnamon). AppImage собирается под glibc ≥ Ubuntu 22.04.
- Bluetooth-адаптер + BlueZ (`bluetoothctl`) — почти всегда уже есть в системе.
- Windows/macOS: best-effort, не покрываются AppImage-сборкой; см. `periprint-spec.md`.

## Устранение проблем

**«Bluetooth-модуль не найден или несовместим с этой сборкой Python»** — не установлен системный пакет `python3-bluez`, либо его версия Python не совпадает с той, под которую собран AppImage:
```bash
sudo apt install python3-bluez
```

**Принтер не находится при сканировании** — убедитесь, что это Bluetooth Classic (SPP), а не BLE-профиль устройства (в списке `bluetoothctl` принтер обычно виден как `PPG_<модель>_<id>`, без суффикса `_BLE`); попробуйте сначала вручную `bluetoothctl scan on`.

**Задание встало на паузу с ошибкой во время печати** — приложение слушает статусы принтера в реальном времени (бумага/крышка/перегрев); устраните причину и нажмите «Продолжить» — задание допечатает с того же места, не с начала.
